# 9 Glossar

Dieses Glossar erklärt Begriffe und Abkürzungen, die im Projekt und in den Projektunterlagen verwendet werden.

AG: Auftraggeber  
AN: Auftragnehmer  
IFE: Inflight-Entertainment-System, Unterhaltungssystem im Flugzeug  
Dev-Kit: Entwicklungs- und Testkit des Auftraggebers, inklusive IFE-Simulator und technischer Dokumentation  
UI: Benutzerschnittstelle der Anwendung  
UX: Nutzererlebnis und Bedienbarkeit  
CI: Corporate Identity, Vorgaben zum Erscheinungsbild einer Airline
FA: Funktionale Anforderungen

Bot: Computergegner im Einzelspielermodus  
PvP: Player versus Player, Spielmodus zwei Spieler gegeneinander  
PvE: Player versus Environment, Spielmodus gegen Bot

Git: Versionsverwaltungssystem  
GitHub: Repository-Plattform für Quellcode, Dokumente und Pull Requests  
Pull Request: Änderungsvorschlag zur Zusammenführung von Änderungen in den Hauptzweig

UML: Unified Modeling Language, Notation zur Modellierung von Systemen  
Papyrus: UML-Tool als Eclipse-Plugin

Maven: Build-Tool für Java-Projekte  
IntelliJ IDEA Community Edition: Entwicklungsumgebung für Java

JUnit 5: Testframework für automatisierte Tests  
Mockito: Bibliothek zum Erstellen von Mock-Objekten für Tests

Markdown: Textformat für Dokumentation im Repository  
Javadoc: API-Dokumentation, die aus Kommentaren im Java-Code als HTML generiert wird  
ProGuard: Werkzeug zur Obfuscation von Java-Anwendungen  
Checkstyle: Werkzeug zur Prüfung von Coding-Style-Regeln
